1. INICIALIZACION DEL REPOSITORIO LOCALMENTE:

Lo principal es tener instalado Git y tener conocimientos previos conocimientos en la Consola o CMD el primer comando que debemos usar para crear este repositorio es Git init, el cual va a crear una carpeta git en el directorio actual esa carpeta tendrá todas Las dependencias y archivos que necesita git para realizar sus procesos.

El comando es: git init Git add o Git add folder o archivo Me permite guardar los cambios realizados en esos archivos o carpetas Git commit-m’name commit´ me permite colocarles un nombre a los cambios Guardados anteriormente es muy potente ya que me permite llevar un versionamiento De proyecto eje, cree un modelo los pasos a seguir para guardarlos serian Git add modelo Commit para hacer la doc o versionamiento de mi modelo Git commit -m ´creación modelo´

Comandos usados

  • Git Init
  • Git add .
  • Git Commit -m 'commit nombre'

2 CREACION DE BRANCHES:

El comando git Branch new_ Branch es para crear una nueva rama, la cual nos va a Permitir el trabajo en equipo, sin dañarnos el trabajo unos a otros, esto nos permite Llevar un mejor orden y nos da la facilidad de tener un mejor trabajo en equipo Ya que si bien es verdad todos vamos a estar trabajando en un mismo proyecto Previamente clonado, cada integrante puede tener una rama para ir trabajando en un Modulo en específico. Es muy potente esta herramienta de las ramas ya que nos Permite unir las modificaciones o cambios en una sola unidad con el comando Git merge Branch

donde Branch es la rama con la que deseo combinar los cambios Es decir, puede haber un conjunto de desarrolladores trabajando en distintos módulos Y luego con este comando git merge puedo ir uniendo los módulos me da la capacidad De que puedo ir depurando mucho mejor los códigos y me ofrece la facilidad de Identificar incompatibilidades al hacer este merge. en pocas palabras une bloques De códigos en uno solo Git Branch new_ Branch Git merge Branch

E je punto 2 Supongamos que tenemos una app de un supermercado, tengo 3 desarrolladores Los cuales están trabajando en distintos módulos uno en la parte de administración Otro en la parte de usuario y otro en los inventarios, se crea un repositorio para todo el Proyecto y a cada uno se le crea una rama para diseñar el módulo encargado. en un Principio el proyecto esta modularizado y con el comando que me ofrece git que es el Merge puedo unir estos segmentos módulos o bloques de código

Comandos usados

  • Git Branch new_ Branch
  • Git merge Branch

Subida a GITHUB

Copiamos la instrucción GITINIT para iniciar nuestro repositorio local, el cual nos crea una carpeta llamada .git la cual contiene todas las librerias o dependencias que git necesita

Usamos el comando git add . para agregar todos los cambios de nuestro fichero a nuestro repositorio local

Usamos git commit para manejar el order o versionamiento de nuestro proyecto. en el cual definimos como la tercera version. git commit normalmente se usa para guardar una version ya sea de una mejora, una imprementación o un manejo de error

En esta instrucción usamos git remote add origin raiz. Para agregar nuestro repositorio local a un repositorio remoto previamente creado en github, esto nos sirve para almacenar nuestro proyecto en la nube de github por así decirlo.

Y llegamos a lo ultimo que es el git push origin main que nos sirve para subir todos los cambios realizados o mejor dicho para subir nuestro commit a la rama específicada que en este caso es la main.
